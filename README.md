# Lumen

Airport pickup-demand intelligence dashboard built with Next.js 15.

## Overview

Lumen analyzes flight arrival data to help anticipate ground transportation demand at airports. It surfaces key metrics (total arrivals, delays, peak windows) and visualizes demand curves so drivers and operators can position ahead of passenger surges.

## Features

- **Analytics overview** — metric cards, hero insight, and demand charts per airport
- **Flights list** — paginated, filterable table of individual arrivals
- Airport selector that scopes all data to a chosen airport

## Tech stack

- [Next.js 15](https://nextjs.org/) (App Router)
- [React 19](https://react.dev/)
- [Tailwind CSS](https://tailwindcss.com/) + [shadcn/ui](https://ui.shadcn.com/)
- [Recharts](https://recharts.org/) for demand visualization
- [TanStack Query](https://tanstack.com/query) for data fetching

## Data

The app works against a single static snapshot (timestamped `2026-05-03T22:00:00Z`). Flight data lives in `src/data/flights.ts` — a TypeScript array auto-generated by `scripts/generate-flights.mjs`; do not edit it by hand.

At runtime, a Next.js API route (`/api/flights`) reads that array, applies filtering/sorting, computes insights, and returns JSON. The client hooks in `src/hooks/` fetch from that route via TanStack Query. There is a simulated 300 ms latency in the route to mimic a real API.

To replace the mock with a live feed, swap the static import in `src/app/api/flights/route.ts` with a real data source — the rest of the stack (route, hooks, UI) stays the same.
